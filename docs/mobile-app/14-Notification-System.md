# JivniCare Patient Mobile App
# Notification System

**Version:** 1.0.0

**Status:** Design Freeze

---

# Purpose

This document defines the complete Notification System for the JivniCare Patient Mobile App.

Notifications are intended to:

- Keep patients informed
- Reduce waiting time
- Improve attendance
- Deliver only meaningful information

Notifications should always provide value.

Users must never feel spammed.

---

# Notification Philosophy

Every notification must satisfy at least one of the following:

- Action Required
- Time Sensitive
- Status Changed
- Important Information

If a notification does not satisfy any of these conditions, it should not be sent.

---

# Notification Objectives

The notification system must:

- Inform users about booking updates
- Keep users updated about queue progress
- Notify users when action is required
- Reduce missed consultations
- Improve patient experience

# Security Rules

Notifications must never expose:

OTP

Authentication Tokens

Internal IDs

Sensitive medical information

Personal identifiers beyond what is necessary

Example

Preferred

"Your booking is confirmed."

Avoid

"Your diabetes consultation booking is confirmed."

---

# Privacy Rules

Notification previews must respect device privacy settings.

Sensitive details should remain hidden when preview settings require it.

# Developer Rules

Developers must never:

Send duplicate notifications unnecessarily.

Expose sensitive information.

Ignore notification priority.

Hardcode notification timing.

Implement client-generated notifications for business events.

Backend remains the source of truth.

---

# Acceptance Criteria

Notification implementation is complete when:

✓ Queue updates notify correctly.

✓ Booking events notify correctly.

✓ Notifications deep link properly.

✓ Read/Unread state is maintained.

✓ Notification settings work.

✓ Sensitive information remains protected.

✓ Users receive timely, actionable notifications.
