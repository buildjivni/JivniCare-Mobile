## Mandatory Rule

> **Always reference assets from `@assets/brand/`. Do not generate or use placeholders for these files.**

- Import or require files from `@assets/brand/` — never invent SVG/PNG paths, never draw
  substitute logos in code, and never commit ad-hoc placeholder images.
- SVG is the master source of truth; PNG variants are generated or exported from those masters.
- For component-to-asset mappings and layout constraints, see the mapping tables in this
  document (Sections "Component Mapping" and "Strict Prohibitions" below).

## Usage Guidelines

### Core Principles
- **Golden Rule:** Branding is functional, not decorative. Every logo or icon placement must
  have a clear identity or navigational purpose. Never place branding merely to fill empty
  space; empty space must remain empty.
- **Consistency Over Creativity:** These rules always override personal design preferences.
  Guessing or making ad-hoc variations is strictly prohibited.

### Asset Integrity & Rendering Rules
- **Master Asset Rule:** SVG is the only master source of truth. All platform-specific PNG
  assets must be programmatically generated from the master SVG to maintain vector sharpness.
- **Safe Scaling Rule:** When scaling assets, aspect ratios must be strictly locked. Safe
  padding defined within the master assets must be preserved. Never stretch, squish, crop,
  rotate, skew, or distort logos.
- **No Placeholders Rule:** Do not use temporary or placeholder logos in any build — this
  reinforces the Mandatory Rule above.
- **No Artificial Container Rule:** Do not enclose logos or brand icons in artificial boundaries
  (custom borders, shadowed boxes, circular wraps) unless explicitly specified in the component
  mapping below.
- **Prohibited Effects:** Never apply shadows, outlines, glow, filters, or opacity modifiers to
  any logo or icon.

### Layout & Positioning Rules
- **Asset-Driven Layout Rule:** The layout must adapt to the brand assets. Never resize or
  distort a logo to fit a poorly designed container; fix the container dimensions to accommodate
  the logo's native aspect ratio.
- **Dynamic Layout Rule:** Layouts containing brand assets must scale elegantly and dynamically
  across mobile, tablet, and desktop viewports, maintaining proper margins and alignment.
- **Duplication Restriction:** Maximum of one branding element per logical section, and one
  Primary Logo per viewport. Never display the Primary Logo and the Brand Icon together in the
  same logical container.
- **Footer Branding Rule:** A footer rendering the Primary Logo must use the official
  brand-blue background (`#5696C7`).
- **Approved App Icon Safe Area:** App icons must preserve an 11% safe-padding margin (matching
  the "Variation 10" safe-area configuration baked into the master app-icon SVG) to prevent
  clipping during platform-specific cropping (circle, squircle, rounded square).

### Component Mapping

| UI surface | Approved asset | Path under `@assets/brand/` | Notes |
|------------|----------------|------------------------------|-------|
| App header | Primary Logo | `primary-logo/primary logo.svg` | Top navigation header |
| Splash / loading screen | Brand Icon | `brand-icon/brand-icon.svg` | Central progress indicator |
| Avatar / profile placeholder | Brand Icon | `brand-icon/brand-icon.svg` | User profile picture placeholder |
| Auth screens (login, signup, OTP) | Primary Logo | `primary-logo/primary logo.svg` | Centered identity placement on auth forms |
| Success screens | Brand Icon | `brand-icon/brand-icon.svg` | Center completion badge |
| Empty states (No Appointments, No Results) | Brand Icon | `brand-icon/brand-icon.svg` | Informational placeholder |
| Error states | Brand Icon | `brand-icon/brand-icon.svg` | Center status decoration |
| Browser tab / bookmark | Favicon | `favicon/favicon-16x16.png` | Programmatically generated PNG sizes |
| App launcher (iOS/Android/PWA) | App Icon | `app-icon/app-icon-1024.png` (or size-appropriate PNG) | Size-appropriate PNG per platform |
| Search / landing / featured-doctor sections | **No branding** | — | Focus stays on search input and content, no decorative logo overlays |

### Strict Prohibitions (No-Branding Zones)

Never embed logos, icons, or wordmarks inside:
- Doctor Cards / Booking Widgets (no decorative logos inside card borders)
- Hospital/Clinic info cards
- Appointment / Queue / Token cards
- Dashboard widgets, charts, analytics tables

### Implementation Verification Checklist

Before shipping any change that places or updates a brand asset:
1. **Visual balance** — padding/margins consistent with the surrounding screen.
2. **Sufficient contrast** — logo variant (default/white/black) matches background contrast.
3. **Responsiveness** — layout behaves correctly across mobile/tablet/desktop breakpoints.
4. **Accessibility** — every image/inline SVG has an `alt` description or `accessibilityLabel`
   per `09-Component-Library.md`'s Shared Accessibility Base.
5. **No placeholders** — the real asset from `@assets/brand/` is wired in, not a stand-in.
