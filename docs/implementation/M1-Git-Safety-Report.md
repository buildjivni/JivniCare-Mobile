# Engineering Safety Check — Git Tracking of `docs/`

**Date:** 2026-07-20
**Trigger:** Pre-Milestone-2 gate check, requested explicitly before any Milestone 2 work begins.
**Scope:** Read-only verification of git tracking status for every file under `docs/`. **No
production code, configuration, or documentation content was modified as part of this check.**

**Result: 🛑 STOP — 7 documentation files are untracked by Git.** See Section 2.

---

## 1. Method

- `git ls-files docs/` — enumerated every file Git currently tracks under `docs/`.
- `Get-ChildItem -Recurse -File docs/` — enumerated every file that actually exists on disk under
  `docs/`.
- Diffed the two lists to find untracked files.
- `git status --ignored --short docs/` and `git check-ignore -v <path>` for each untracked
  file — to distinguish "untracked because never `git add`ed" from "untracked because a
  `.gitignore` rule excludes it."
- Read `.gitignore` in full and checked `.git/info/exclude` for any local-only exclude rules.

### 2.2 — Is any engineering document ignored by `.gitignore`? **NO.**

`.gitignore` was read in full (see below) — it contains no pattern matching `docs/`,
`docs/engineering/`, or any `*.md` file. `git check-ignore -v` returned no match (exit code `1`,
"not ignored") for every untracked file listed above. `.git/info/exclude` is the untouched Git
default template and contains no active rules either.

**Conclusion: the 7 files above are untracked, but not *ignored*.** They were simply never
`git add`ed/committed — a distinct condition from being excluded by `.gitignore`. This is a lower
severity than "actively ignored," but it is functionally the same risk: these files would not be
included in a `git push`, and are invisible to anyone who only inspects tracked history (e.g. a
fresh `git clone`).
